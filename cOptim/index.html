Introduction

This project is a C++ translation of SciPy's global optimization algorithms. Currently supported functions are anneal, basinhopping, and brute.

coptim::anneal(anneal_params &params)

coptim::basinhopping(basin_params &params)

If all DOFs are tested at the same points: coptim::brute(obj_func func, int num_dofs, const range &rng, bool disp)

If each DOF range needs to be specified individually: coptim::brute(obj_func func, vect_states &input, bool disp)

Dependencies

This library requires a C++11 compatible compiler. Due the dependency on dlib for local minimization algorithms, objective functions should be written to take a dlib::matrix<double, 0, 1> as input and output a double. This dependency might be lifted in future versions. In order to locate the dlib source files, the environment variable DLIB_HOME must be set to the dlib source directory.

Usage

To use this library clone the repository and download dlib. In your project link to the cOptim subfolder and the dlib library. Then write your objective function to accept a dlib::matrix<double, 0, 1> and return a double and optimize using the appropriate global optimization technique.

An example for the Rosenbrock function is shown below:

#include "cOptim.h"
//Within cOptim there are standard typdefs that are used frequently in the library
//dlib::matrix<double, 0, 1> is refered to as a column_vector, i.e.
//typedefs.h defines: typedef dlib::matrix<double, 0, 1> = column_vector
//With this in mind let's first define the objective function
double rosen(const coptim::colum_vector &x) {
const double x = m(0);
const double y = m(1);
//compute Rosenbrock's function and return the result
return 100.0*pow(y - x*x, 2) + pow(1 - x, 2);
}
//Now define an initial starting point
coptim::column_vector x0(2);
x0 = 9, 9; //refer to the dlib documentation for the properties of thier matrix implementation
//initialize parameters for the basinhopping algorithm and change
//the initial step size to 0.1 and set disp = true. Note, this library uses the
//named parameter idiom for setting specific parameters for optimization
//in the basinhopping and anneal algorithms.
coptim::basin_params params = coptim::basin_params(rosen, x0)
.step_size(0.1)
.disp(true);
//Then call the basinhopping algorithm
OptimResult res = coptim::basinhopping(params);
//the result is in the form of a struct that has members x, cost, message, success, and niter
//described in the OptimResult section of the documentation. to view the results call res.print()
res.print();
